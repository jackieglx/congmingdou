# 聪明豆项目 · 协作规矩

本文件是本项目以后所有改动都要遵守的规矩。每次动手前请先读一遍。

## 文件结构
- **index.html**：页面结构（各 `screen-*` 界面骨架）。
- **app.js**：所有逻辑 / 函数（约 1.2 万行，游戏、识字测验、错字本、渲染等都在这里）。
- **data.js**：所有静态数据（字库、识字表 `EXAM_BANK`、看字选图题库 `WORDPIC_BANK`、绘本等）。
- **styles.css**：所有样式，配色变量定义在 `:root`。

## 绝对红线
- `data.js` 里的题库 / 字库 / 识字表 / 绘本等「学习内容」属于**教学数据**：任何任务都**不许翻译、改动、增删**其中的字和拼音——除非任务**明确就是在改这份数据**。
- **题面汉字**（识字测验的字、各游戏的题目字、绘本正文）属于**内容、不是 UI**，永远保持**纯中文**，不加英文、不翻译。
- 拿不准某段中文是「UI 文字」还是「题目内容」时：**跳过、不改**，并向用户提出来确认，绝不擅自猜测。
- **错字本只有一套**：需要记录错字时，必须复用现有的 `addWrongChar(char, pinyin, meaning, source, lesson)` 函数（它本身按字去重），数据存在 `czmd_wrong_chars_<username>`。绝不另建第二套错题本或另起一套存储。

## 每次改动的工作方式
- **一次只做一件事 / 一个界面**：做完就报告，等用户验证后再做下一件。不要一轮里铺开改很多处。
- **改前先定位**：用 grep / 读代码找到目标，把计划说清楚；不确定的先问，不要猜。
- **纯新增优先**：不重命名、不重构、不顺手「优化」无关代码。

## 改完必须自查
- 对改过的 `app.js` / `data.js` 跑 `node --check`，确认无语法错误。
- 列出具体改了**哪些文件、哪些行**，并给出**原文 → 改后**。
- 明确确认**没有触碰红线里的教学数据**。

## UI / 双语 / 样式
- **凡是 UI 文字都要中英双语**：所有按钮、菜单、标题、说明 / 指引（instructions）、提示语、反馈语、状态标签（如分数、计数、第几遍 / 第几题）等，只要不是题目本身，都必须同时有中文和英文。
- **题目内容绝不双语**：识字测验的字、各游戏的题目汉字、绘本正文等保持纯中文。
- **已经有英文的地方不要重复加**：先看现状，只给「只有中文」的地方补英文。
- **双语格式统一**：用「中文 · English」横排，或中文在上、英文小字在下，跟现有风格一致，不自创新样式体系；新加的英文要简短，别把小按钮 / 小标签撑长撑乱。
- **必须同时在电脑和手机上都美观**：手机窄屏（约 360–414px）下不挤、不溢出、不错位、不换行错乱、不被截断、不遮挡内容。加了英文若窄屏会挤，用现有 CSS 手段（字号 / 间距 / 必要的 @media）解决，沿用 `styles.css` 既有变量（`--ink` / `--blue` / `--radius` 等）和写法，不引入与全站不一致的新样式。
- **不破坏现有美感和画风**：配色、圆角、阴影、卡片风格都跟随现有 UI，改动只做必要的，不擅自重新设计。

## 部署 / 安全
- 改完**不要让用户删 `index.html.bak`**（备份文件）。
- 由用户用 `firebase hosting:channel:deploy test` 部署到**预览频道**验证后，才推正式线上。
